The RMagick library is released under the MIT license, which can be found at the end of this document. ImageMagick uses a GPL compatible license (see the ImageMagick website).

Prerequisites

The 1.14.1 distribution comes with some changes to the prerequisites. The .so file is no longer statically linked to the ImageMagick library. Instead, you will need to install ImageMagick separately. (See Installation instructions below). A copy of the ImageMagick 6.3.0-7 Q8 installer is included with this package. Do not use any other version of ImageMagick.

(One of the nice parts of having to use the ImageMagick installer is that postinstall.rb isn't totally necessary. In fact, it had to be altered to remove the setting of the MAGICK_CONFIG_PATH environmental variable.

(Unfortunately, this also may mean that you will need to use full pathnames for Windows fonts.)

The RMagick library was built against Ruby version ruby 1.8.5 (2006-08-25) [i386-mswin32]. This is the version of Ruby included in the One Click Installer, version 1.8.5-25.

The gem was packaged using RubyGems version 0.9.4.

The binary .so was built using the VC++ 2003 Toolkit, as described at the Ruby Garden wiki. This compiler may have incompatibilities with the VC++ 6 compiler used to build Ruby for the One Click Installer. No guarantees are made in the release of this software.

This distribution comes in the form of a RubyGem, since I firmly belive that is the way to go for distribution of Ruby libraries. You should have a working copy of RubyGems, which can be downloaded from rubygems.rubyforge.org.

Oh, and erm..

    gem --version

should yield

    0.9.0

or higher. Just do that gem update --system once in a while.

Uninstalling of old versions

Removal of RMagick win32 1.5.0 (ImageMagick 6.0.0) or higher

The old gem should not bother you, since RubyGem does the right thing. However, you can always:

    gem uninstall RMagick

or (for older versions of rubygems)

    gem --uninstall RMagick

Don’t forget to update your version of rubygems.

You get

    Attempting to uninstall gem 'rmagick'
    Successfully uninstalled RMagick-win32 version 1.6.0

or the like.

You will also need to remove the MAGICK_CONFIGURE_PATH entry in the Windows registry. To do so manually, use:

    Start > Run... > regedit
Click on "My Computer" in the sidebar on the left. Before altering your registry (which you may regret at some point, particularly if you accidentally change or delete something), you may want to make a back up. To do so, from the menus choose 
    File > Export...
choose a filename, and save a copy. Once you've done this (or if you choose not to), from the menu, select 
    Edit > Find...
Then enter "MAGICK_CONFIGURE_PATH". If that entry contains an RMagick path, eg 
    C:\ruby\lib\ruby\gems\1.8\gems\rmagick-1.13.0-win32
Then delete it.

Removal of RMagick win32 1.4.0

Those who have RMagick working in its version 1.4.0 should carefully remove it by hand (sorry). This involves:

  • deleting RMagick.so from site_ruby/i386/
  • deleting RMagick.rb from site_ruby
  • deleting the registry key called ImageMagick

Mixing the two distributions will not succeed and will probably result in crashes.

Installation instructions

Gem Install

The Zip archive you have downloaded should contain the following files:

File Description
README.html the file you are reading
README-RMAGICK.HTML original RMagick Readme (HTML)
README-RMAGICK.txt original RMagick Readme (Text)
ImageMagick-6.3.0-7-Q8-windows-dll.exe ImageMagick installer
RMagick-1.14.1-win32.gem the gem

Let’s go !

First, use the ImageMagick installer found in the zip file. The only option that you need to choose from the installation options is "Update executable search path." The other options are up to you. Do not use any other version of ImageMagick. If you have problems with RMagick we can help only when you are using the version of ImageMagick that is bundled with this release of RMagick.

Installing the gem is very simple:

    gem install rmagick --local

You should see:

    Successfully installed rmagick, version 1.14.1

Post Installation

It may be necessary to restart any services, cmd.exe sessions, or even reboot for current applications (Ruby on Rails based apps, in particular) to use a fresh upgrade.

Documentation

Documentation has been installed into the directory of the gem, which can be found using the command:

    gem environment
The line(s) you should note is/are: 
    - GEM PATH:
       - c:/ruby/lib/ruby/gems/1.8
Using this example, the documentation can be found in: 
    c:\ruby\lib\ruby\gems\1.8\gems\rmagick-1.14.1-win32\docs
And the examples are in: 
    c:\ruby\lib\ruby\gems\1.8\gems\rmagick-1.14.1-win32\examples

General Remarks

If Ruby complains that it can't find RMagick, be sure that you have set up the RubyGems environment. The simplest way to do this is to require "rubygems" first:

        require 'rubygems'
        require 'RMagick'
There are other ways to set up the RubyGems environment. See the RubyGems documentation for more information.

Use require 'RMagick’, spelled as shown. It seems that we managed to sneak a bit of case sensitivity into Ruby.

Don't use gem to require RMagick. The gem statement is used only when you want to require a specific version of a library.

Help If you need help installing or using RMagick, use the Help Forum on RubyForge.

Bugs and Comments Please report bugs in RMagick, its documentation, or its installation programs to us via the bug tracker on the RMagick project page at RubyForge. When reporting bugs, please specify that you are using the binary distribution of RMagick for windows. Don’t forget to mention Ruby and Windows version.
